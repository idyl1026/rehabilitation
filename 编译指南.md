# 病程自动生成系统 - 编译指南

## ⚠️ 重要说明

由于您当前使用的是macOS系统，无法直接编译Windows exe文件。请按照以下步骤在Windows电脑上完成编译。

---

## 🖥️ 在Windows电脑上编译

### 准备工作

1. **确保Windows电脑安装了以下软件：**
   - [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
   - （推荐）[Visual Studio 2022](https://visualstudio.microsoft.com/downloads/)

2. **复制项目文件夹**
   ```
   将整个 MedicalProgress 文件夹复制到Windows电脑上
   ```

---

## 📝 方法一：使用Visual Studio编译（推荐）

### 步骤：

1. **打开解决方案**
   - 打开Windows文件资源管理器
   - 进入 `MedicalProgress` 文件夹
   - 双击 `MedicalProgress.sln` 文件
   - Visual Studio会自动启动

2. **编译项目**
   - 在Visual Studio菜单中选择 `生成` → `重新生成解决方案`
   - 等待编译完成（底部状态栏会显示进度）

3. **发布exe文件**
   - 在解决方案资源管理器中右键点击 `MedicalProgress.App`
   - 选择 `发布`
   - 选择发布目标：`文件夹`
   - 设置目标文件夹：`..\publish`
   - 点击 `发布` 按钮

4. **找到exe文件**
   ```
   MedicalProgress\publish\MedicalProgress.App.exe
   ```

---

## 📝 方法二：使用命令行编译

### 步骤：

1. **打开命令提示符**
   - 按 `Win + R`，输入 `cmd`，回车
   - 进入项目目录：
     ```cmd
     cd C:\您的路径\MedicalProgress\src\MedicalProgress.App
     ```
     （请根据实际路径调整）

2. **编译并发布**
   - 输入以下命令：
     ```cmd
     dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o ..\..\publish
     ```

3. **等待完成**
   - 命令执行成功后，会在 `MedicalProgress\publish` 目录生成 `MedicalProgress.App.exe`

---

## 📝 方法三：使用发布脚本（最简单）

1. **双击运行**
   ```
   MedicalProgress\发布.bat
   ```

2. **等待自动完成**
   - 脚本会自动完成所有编译步骤

3. **查看结果**
   ```
   MedicalProgress\publish\MedicalProgress.App.exe
   ```

## 🐛 常见错误及解决方法

### 错误1：找不到 dotnet 命令
**原因**：未安装.NET SDK或未配置环境变量

**解决**：
1. 下载并安装 [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
2. 重启命令提示符或Visual Studio

### 错误2：无法访问路径
**原因**：路径包含中文或特殊字符

**解决**：
1. 将项目移到纯英文路径
2. 例如：`C:\Projects\MedicalProgress`

### 错误3：缺少依赖包
**原因**：NuGet包未恢复

**解决**：
```cmd
dotnet restore
dotnet build
```

### 错误4：编译失败
**解决**：
1. 查看错误信息
2. 确保安装了正确版本的Visual Studio或.NET SDK
3. 尝试清理后重新编译：
   ```cmd
   dotnet clean
   dotnet restore
   dotnet build
   ```
