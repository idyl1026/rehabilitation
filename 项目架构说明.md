# 病程辅助书写与出院病例科研库 - 项目架构说明

## 1. 产品定位

本软件从“简单病程模板生成器”调整为“病程辅助书写 + 出院病例结构化归档 + 科研数据检索导出”的本地桌面工具。

核心目标：

- 根据患者当前病情变化、诊疗节点和既往病程，推荐合适的病程模板。
- 自动把患者基本信息、诊断、检查、治疗、病情变化填入模板，减少重复书写。
- 保留医生编辑、确认和质控环节，软件只做辅助，不替代临床判断。
- 出院后将病例沉淀为可查询、可脱敏、可导出的科研数据库。
- 保持当前技术栈：`.NET 8.0`、`Windows Forms`、`SQLite`、`Entity Framework Core`、`OpenXML`。

## 2. 设计原则

1. 临床书写优先：主界面围绕“今天这个患者发生了什么变化，该写哪类病程”组织。
2. 模板推荐可解释：每次推荐都显示命中依据，例如诊断、病程阶段、关键事件、异常指标。
3. 结构化数据优先：关键临床变量进入数据库字段，长文本保留原文，便于后续科研。
4. 医生最终确认：生成内容必须经医生预览、修改、保存，不能自动落地为正式病历。
5. 科研使用脱敏：导出科研数据时默认移除姓名、住院号、床号、联系电话等直接标识。
6. 本地优先、可扩展：先用规则和本地模板完成闭环，后续可接入院内系统或大模型服务。

## 4. 功能模块调整

### 4.1 病情变化采集

新增 `ClinicalChangeForm`，用于记录本次病程的触发原因和变化要点。

建议采集字段：

| 字段 | 说明 |
| --- | --- |
| ChangeType | 病情变化类型：稳定、好转、加重、术后、检查回报、用药调整、会诊后、出院前等 |
| EventTime | 变化发生时间 |
| Symptoms | 新发或缓解症状 |
| VitalSigns | 体温、血压、心率、呼吸、SpO2 等 |
| ExamFindings | 查体变化 |
| LabHighlights | 关键化验变化 |
| ImagingHighlights | 影像/电生理/功能检查要点 |
| TreatmentChanges | 治疗调整 |
| RiskFlags | 高风险标记：感染、出血、跌倒、VTE、吞咽风险、压疮等 |
| DoctorAssessment | 医生判断 |
| NextPlan | 下一步计划 |

这些字段既用于模板推荐，也用于科研数据结构化。

### 4.2 模板推荐

新增 `TemplateRecommendationService`，把“模板选择”从手工下拉框升级为推荐列表。

推荐输入：

- 患者基本资料：年龄、性别、科室、诊断、入院天数、是否出院。
- 病程上下文：首次病程、日常病程、上级医师查房、阶段小结、抢救记录、术后/治疗后记录、出院前记录。
- 本次病情变化：`ClinicalChange` 中的变化类型、风险标记、检查结果、治疗调整。
- 历史病程：最近一次病程、重复内容检测结果。
- 知识库模板：模板类型、疾病分类、关键词、适用条件、使用次数。

推荐输出：

| 字段 | 说明 |
| --- | --- |
| TemplateId | 推荐模板 |
| Score | 推荐分数 |
| Reason | 推荐理由 |
| MatchedSignals | 命中信号，例如“脑卒中 + 吞咽风险 + 康复治疗调整” |
| MissingData | 生成前建议补充的数据 |
| RiskNotice | 风险提醒，例如“需确认抗凝禁忌证” |

推荐规则优先级：

1. 强规则：病程阶段和医疗事件必须匹配，例如出院办理只能推荐出院小结类模板。
2. 疾病规则：诊断、疾病分类、关键词匹配。
3. 变化规则：加重、好转、检查回报、治疗调整等事件匹配。
4. 质量规则：避免推荐近期已连续使用且内容高度重复的模板。
5. 使用反馈：医生常用、最近修改、保存成功率较高的模板适当加权。

### 4.3 病程生成

保留现有 `GeneratorService`，但职责调整为“根据推荐模板 + 结构化变化 + 患者上下文生成草稿”。

生成流程：

```text
选择患者
  → 填写/导入本次病情变化
  → TemplateRecommendationService 给出模板候选
  → 医生选择模板
  → GeneratorService 填充患者信息、病情变化、既往病程摘要
  → QualityControlService 检查缺项、重复、日期、诊断一致性
  → 医生编辑确认
  → 保存 ProgressRecord
```

模板变量建议统一为：

```text
{{患者.姓名}}
{{患者.年龄}}
{{患者.性别}}
{{患者.住院号}}
{{患者.入院诊断}}
{{病程.记录日期}}
{{变化.类型}}
{{变化.症状}}
{{变化.生命体征}}
{{变化.检查结果}}
{{变化.治疗调整}}
{{变化.医生评估}}
{{变化.下一步计划}}
{{既往病程.最近摘要}}
```

### 4.4 出院病例归档

新增 `ArchiveService` 和 `DischargeArchiveForm`，把出院动作拆成两个层次：

- 临床出院：生成出院小结、出院医嘱、Word 文档。
- 科研归档：生成结构化病例快照，进入 `CaseArchives` 表。

归档内容：

| 模块 | 内容 |
| --- | --- |
| 基本信息 | 年龄、性别、科室、入院/出院日期、住院天数 |
| 诊断信息 | 入院诊断、出院诊断、主要诊断、合并症 |
| 诊疗过程 | 主要治疗、康复治疗、药物、手术/操作 |
| 检查摘要 | 关键化验、影像、量表、功能评估 |
| 转归 | 好转、未愈、转院、死亡等 |
| 病程全文 | 可保留原文，供回顾性查看 |
| 结构化标签 | 疾病标签、治疗标签、风险标签、科研标签 |
| 脱敏字段 | 研究编号、脱敏状态、导出批次 |

归档状态：

- `Draft`：出院小结已生成，但未确认归档。
- `Archived`：医生确认后进入科研病例库。
- `Locked`：归档后锁定临床原始快照，后续修改产生新版本。
- `Excluded`：病例不纳入科研库，但仍保留临床文档。

### 4.5 科研病例库

新增 `ResearchQueryForm`，用于筛选、队列管理和导出。

基础筛选：

- 时间范围：入院日期、出院日期。
- 人群条件：年龄范围、性别、科室。
- 疾病条件：入院诊断、出院诊断、疾病分类、关键词。
- 治疗条件：药物、康复治疗、手术/操作。
- 结局条件：住院天数、转归、再入院标记。
- 文本检索：主诉、现病史、病程、出院小结全文关键词。

科研队列：

- 保存查询条件为 `ResearchCohort`。
- 支持病例加入/移出队列。
- 支持队列备注和纳排标准。
- 支持导出 CSV、Excel、JSON。

导出默认策略：

- 默认脱敏导出。
- 导出前显示字段清单。
- 每次导出记录 `ResearchExportJob` 和 `AuditLog`。
- 原始身份信息只能在临床模块查看，科研模块默认不显示。

## 8. 服务职责调整

### ProgressAssistantService

病程辅助总入口，编排病情变化、模板推荐、生成、质控和保存。

核心方法：

- `CreateDraft(patientId, clinicalChangeId)`
- `GenerateFromRecommendation(patientId, recommendationId)`
- `SaveConfirmedProgress(record)`

### TemplateRecommendationService

负责模板召回、评分、排序和解释。

核心方法：

- `Recommend(patient, clinicalChange, records)`
- `ExplainRecommendation(template, matchedRules)`
- `GetMissingFields(template, clinicalChange)`

### ClinicalChangeService

负责病情变化 CRUD 和摘要生成。

核心方法：

- `SaveChange(change)`
- `GetRecentChanges(patientId)`
- `BuildChangeSummary(change)`

### QualityControlService

负责病程保存前检查。

检查项：

- 内容为空或过短。
- 与历史病程重复。
- 日期、住院天数、出院状态异常。
- 模板变量未填充。
- 必填信息缺失。
- 诊断与模板疾病分类明显不匹配。

### ArchiveService

负责出院病例归档。

核心方法：

- `BuildArchiveDraft(patientId)`
- `ConfirmArchive(caseArchiveId)`
- `LockArchive(caseArchiveId)`
- `RebuildArchiveSnapshot(patientId)`

### ResearchDatasetService

负责科研检索、队列和导出。

核心方法：

- `SearchCases(criteria)`
- `CreateCohort(criteria)`
- `AddCaseToCohort(cohortId, archiveId)`
- `ExportCohort(cohortId, fields, deIdentify)`

### DeIdentificationService

负责脱敏。

默认移除或替换：

- 姓名
- 住院号
- 床号
- 身份证号
- 电话
- 详细住址
- 精确日期可按研究需要转换为相对时间或年月

## 9. 业务流程

### 9.1 日常病程辅助书写

```text
1. 选择患者
2. 点击“记录变化”
3. 填写本次病情变化
4. 系统推荐模板并显示推荐理由
5. 医生选择模板
6. 系统生成病程草稿
7. 系统执行质控检查
8. 医生编辑、确认
9. 保存病程记录
```

### 9.2 出院归档

```text
1. 选择患者
2. 点击“办理出院”
3. 填写出院诊断、出院医嘱、转归
4. 生成出院小结和 Word 文档
5. 系统生成科研归档草稿
6. 医生补充主要诊断、治疗标签、结局标签
7. 脱敏预览
8. 确认归档并锁定快照
```

### 9.3 科研检索导出

```text
1. 打开科研病例库
2. 设置纳排条件
3. 查看病例列表和摘要
4. 保存为研究队列
5. 选择导出字段
6. 脱敏预览
7. 导出 CSV/Excel/JSON
8. 记录导出审计
```

## 11. 安全与合规边界

- 软件定位为临床文书辅助工具，不提供诊断结论，不自动改变医嘱。
- 所有生成文本均为草稿，必须由医生确认保存。
- 科研导出默认脱敏，并记录导出日志。
- 原始病例数据与科研导出数据分层管理。
- 如接入外部 AI 服务，必须先完成数据脱敏或使用院内合规环境。
- 删除病例和解锁归档应记录审计日志。

## 13. 资料导入整理模块

本阶段新增“检查资料导入整理”的最小可用闭环，先不依赖本地大模型，适合医院内网和低配置电脑。

已落地的核心对象：

- `ImportedDocument`：保存导入资料原文、清洗后文本、来源文件、资料类型、所属患者。
- `StructuredExamResult`：保存结构化检查结果，包括检查日期、检查类型、项目名、结果、单位、参考范围、异常标记、结论、原始行。
- `DepartmentProfile`：预留科室配置包模型，用于不同专业科室维护诊断、检查、量表、治疗关键词。

已落地的核心服务：

- `TextExtractionService`：从 `txt`、`docx`、`pdf` 中提取文本。PDF 当前为轻量级尽力提取，不引入重型依赖。
- `ClinicalTextNormalizeService`：统一换行、空格、中文/英文标点，去除明显页码噪声。
- `ExamResultParserService`：按规则识别化验、影像、量表，并抽取项目名、结果、单位、参考范围、异常标记。
- `DocumentImportService`：统一复制文本和文件导入入口，返回可预览、可修改的导入结果。

已落地的界面：

- `ImportReviewForm`：支持粘贴剪贴板文本、打开 `txt/docx/pdf` 文件，显示原文、清洗文本和结构化结果表格。医生确认和修正后再保存入库。
- `MainForm`：新增“导入检查资料”入口，选中患者后可直接打开导入确认界面。

当前设计取舍：

- 不强制内置本地模型，避免低配内网电脑运行困难。
- 先用规则、词典、人工确认完成高频检查结果整理。
- `docx` 支持较稳定；老式 `.doc` 暂不直接解析，建议另存为 `.docx` 或通过后续插件转换。
- PDF 因扫描件、加密、字体编码差异较大，当前只做轻量文本型 PDF 的尽力提取；扫描 PDF 后续可作为 OCR 插件扩展。

下一步建议：

1. 为康复科先建立第一版 `DepartmentProfile` 配置，包括常用诊断、评定量表、治疗项目和科研字段。
2. 增加“导入结果用于生成病程”的按钮，把最近检查结果自动插入病程草稿。
3. 增加 `ResearchQueryForm` 对 `StructuredExamResult` 的检索和导出。
4. 若实际资料中扫描 PDF 很多，再单独加入离线 OCR 或院内服务接口。

## 14. 病情分析建议引擎

本阶段新增第一版内置病情分析建议能力，采用“规则 + 模板语句 + 医生确认”的方式，不依赖本地大模型。

已落地对象：

- `AnalysisSuggestion`：分析建议结果，包含分类、标签、依据、分析语句、建议、风险等级。

已落地服务：

- `ClinicalChangeAnalyzerService`：根据病史查体段落、结构化检查结果和清洗文本生成病情分析草稿。

当前内置规则范围：

- 通用化验规则：血红蛋白偏低、红细胞偏低、血钾偏低、D-二聚体升高、CRP 升高。
- 影像规则：颅脑 CT/MRI 与脑梗死恢复期、软化灶、缺血性改变相关；未见新发病灶时提示继续康复治疗基础较稳定。
- 康复科规则：偏瘫功能障碍、远端肌力较差、Brunnstrom 分期、Ashworth/痉挛、平衡能力受限、吞咽呛咳风险观察。
- 一般情况规则：神清、精神可、生命体征平稳、主诉摘要。

已落地界面：

- `ImportReviewForm` 新增 `Analysis draft` 区域。
- 导入资料后自动生成分析草稿。
- `Generate analysis` 按钮可在医生修正结构化结果后重新生成分析。

维护建议：

- 第一版规则先作为代码内置，便于快速验证。
- 后续应迁移到 `DepartmentProfile` 或独立规则表，字段包括触发条件、分析标签、模板语句、建议、风险等级、适用科室、是否启用、维护人、维护时间。
- 康复科规则建议先用 20-50 份真实病程样本校准，再扩展神经内科、骨科、呼吸科等科室规则包。
