# 知识库使用说明

本文档说明知识库的操作方式、模板文件格式规范，以及如何在联网电脑上用 AI 批量生成模板再导入内网电脑。

---

## 一、界面操作

### 主要按钮

| 按钮 | 功能 |
|------|------|
| 新建学科 | 创建新的学科分组（如"康复医学科""神经内科"） |
| 新建模板 | 在当前学科下手动新建单条模板 |
| 编辑模板 | 修改已选中的模板 |
| 删除模板 | 将选中模板标记为不活跃（保留数据库记录） |
| **导入模板** | 从 JSON 或 CSV 文件批量导入模板（主要入口） |
| 导出示例 | 导出示例 JSON / CSV 文件，可用于告知 AI 格式 |
| 刷新 | 重新加载学科和模板列表 |

### 套用模板

**双击列表中的模板** → 内容自动填入病程编辑器，关闭知识库窗口。

### 模糊搜索

搜索框支持多关键词，**用空格分隔**，系统要求所有词同时命中，按相关度排序。

- `脑梗死 首次` → 找标题/关键词/内容同时含"脑梗死"和"首次"的模板
- `康复 吞咽` → 找同时提到两个词的模板
- 标题命中 > 关键词命中 > 摘要命中 > 正文命中（权重递减）

## 二、模板文件格式规范

推荐使用 **JSON 格式**，支持多行内容，AI 生成准确率高；也兼容 CSV。

### 2.1 JSON 格式（推荐）

```json
[
  {
    "title": "模板标题（必填，不超过100字）",
    "type": "日常病程",
    "category": "神经系统",
    "keywords": "脑梗死,偏瘫,Brunnstrom",
    "content": "正文内容（必填，支持换行）"
  },
  {
    "title": "第二条模板",
    ...
  }
]
```

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| title | 是 | 模板名称，建议包含"疾病 + 类型"，如"脑梗死首次病程记录" |
| type | 否 | 模板类型，见下方枚举，默认"日常病程" |
| category | 否 | 分类名称，同名自动归组，建议使用统一命名 |
| keywords | 否 | 逗号分隔关键词，影响搜索命中率，建议填写 |
| content | 是 | 模板正文，换行直接写即可 |

**type 枚举（固定值）：**

```
首次病程    日常病程    出院小结    会诊记录
检查分析    专科评定    抢救记录    手术记录
```

**category 建议命名（可自定义，统一即可）：**

```
神经系统    骨关节    心肺    内分泌代谢    消化    泌尿
康复评定    检查报告    治疗方案    其他
```

---

### 2.2 CSV 格式（备选）

```csv
标题,类型,分类,关键词,内容
脑梗死日常病程,日常病程,神经系统,"脑梗死,偏瘫","患者今日一般情况可，精神食欲正常，继续康复治疗。"
```

- 第一行必须为表头（含"标题"和"内容"列，其余可选）
- 多行内容不便放入 CSV，建议用 JSON
- 文件编码 UTF-8

## 三、用 AI 批量生成知识库模板

### 3.1 工作流程

```
联网电脑（有网）:
  1. 打开 ChatGPT / Claude / 其他 AI
  2. 粘贴下方提示词，附上疾病列表
  3. AI 输出 JSON 文本
  4. 保存为 xxx.json（UTF-8 编码）
  5. 拷贝到 U 盘

内网电脑（无网）:
  6. 打开病程助手 → 知识库管理
  7. 选择目标学科
  8. 点击「导入模板」→ 选择 JSON 文件
  9. 确认导入结果
```

---

### 3.2 AI 提示词模板

以下提示词可直接复制粘贴给 AI，根据需要修改括号内的内容：

---

**【提示词】**

```
你是一名医疗文书专家，请帮我生成病程记录知识库模板，严格按照下方 JSON 格式输出，不要输出任何多余文字。

JSON 格式要求：
- 输出一个 JSON 数组，每个元素是一条模板
- 每条模板包含字段：title、type、category、keywords、content
- type 只能是以下之一：首次病程、日常病程、出院小结、会诊记录、检查分析、专科评定
- keywords 用逗号分隔
- content 是完整的病程模板正文，使用换行符分段，语言规范、符合中国临床书写要求

请生成以下模板：

学科：【康复医学科】
疾病：【脑梗死恢复期】
需要的类型：
1. 首次病程记录（含病例特点、诊断依据、鉴别诊断、诊疗计划）
2. 日常病程记录 - 病情好转版本
3. 日常病程记录 - 病情平稳版本
4. 日常病程记录 - 出院前小结版本
5. 出院小结

模板要求：
- 留出需要医生填写的空白处，用【】标注，如【XX】【肌力X级】
- 包含常用康复评定量表占位符（Brunnstrom、Ashworth、Barthel等）
- 语言简洁，符合三甲医院病历书写规范
- 每条模板正文不少于200字
```

---

可以替换的部分：
- `学科`：康复医学科 / 神经内科 / 骨科 / 内科等
- `疾病`：脑梗死 / 颈椎病 / 脊髓损伤 / 帕金森 / 骨折等
- `需要的类型`：按实际需要列出
- `模板要求`：根据科室规范添加特殊要求

---

### 3.3 常用疾病建议列表

**神经系统（康复医学科 / 神经内科）**
- 脑梗死恢复期、脑出血恢复期
- 蛛网膜下腔出血恢复期
- 脊髓损伤（完全 / 不完全）
- 吉兰-巴雷综合征
- 帕金森病
- 颅脑外伤恢复期

**骨关节（骨科 / 康复医学科）**
- 颈椎病（神经根型 / 脊髓型）
- 腰椎间盘突出症
- 膝关节骨性关节炎
- 肩关节周围炎
- 全髋 / 全膝关节置换术后康复
- 骨折术后康复

**其他专科**
- 慢阻肺（呼吸内科）
- 心力衰竭（心内科）
- 糖尿病足（内分泌科）

### 3.4 导入前核对建议

导入前建议用文本编辑器（如 VS Code / Notepad++）打开 JSON 文件：
1. 确认 JSON 格式正确（无多余逗号、括号匹配）
2. 抽查 2-3 条模板正文，确认内容质量
3. 确认文件编码为 UTF-8（避免中文乱码）

## 四、模板内容书写建议

### 占位符约定

建议统一用【】标注需要医生填写的内容：

```
患者，【男/女】，【XX】岁，主因"【主诉】"入院。
患侧上肢肌力【X】级，Brunnstrom分期上肢【X】期/手【X】期/下肢【X】期。
站立平衡【X】级，改良Ashworth肌张力【X】级。
```

### 段落结构建议

**首次病程（参考）：**
```
一、病例特点
二、诊断及鉴别诊断
三、诊疗计划
```

**日常病程（参考）：**
```
一般情况描述 → 主诉变化 → 查体要点 → 今日治疗 → 病情分析 → 下一步计划
```

**出院小结（参考）：**
```
入院情况 → 诊疗经过 → 出院情况 → 出院诊断 → 出院医嘱
```

## 五、常见问题

**Q：导入失败，提示"JSON 格式错误"**
A：用在线 JSON 校验工具（jsonlint.com）检查格式，常见问题：内容中的引号未转义、多余逗号、括号不匹配。

**Q：内容显示乱码**
A：保存 JSON 文件时选择 UTF-8 编码（不带 BOM）。

**Q：如何批量替换某个词？**
A：先导出所有模板（未来功能），用文本编辑器批量替换后重新导入；目前可在"编辑模板"界面逐条修改。

**Q：同名模板会重复导入吗？**
A：目前不做重名检查，每次导入均新建条目；若重复导入，可在列表中手动删除重复项。

**Q：AI 生成的内容需要改吗？**
A：AI 生成的模板为参考草稿，务必由临床医生审核后再使用，确保内容符合本科室实际规范。
