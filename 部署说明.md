# 病程自动生成系统 - 部署指南

## 📋 系统要求

- **操作系统**：Windows 10/11（64位）
- **运行环境**：.NET 8.0 Runtime（如果使用依赖框架版本）
- **磁盘空间**：约 200MB

---

## 🚀 快速部署

### 方法一：使用发布脚本（一键部署）

1. **复制项目文件夹**
   ```
   将整个 MedicalProgress 文件夹复制到您的Windows电脑
   ```

2. **运行发布脚本**
   ```
   双击运行：发布.bat
   ```

3. **等待编译完成**
   - 脚本会自动下载依赖
   - 编译项目
   - 生成独立的exe文件

4. **找到exe文件**
   ```
   MedicalProgress\publish\MedicalProgress.App.exe
   ```

### 方法二：手动编译

如果您熟悉命令行，可以手动执行以下步骤：

1. **打开命令提示符（CMD）**
   ```
   cd MedicalProgress\src\MedicalProgress.App
   ```

2. **编译并发布**
   ```
   dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o ..\..\publish
   ```

✅ 单文件部署，便于分发

## 🗄️ 数据存储位置

首次运行exe时，系统会在以下位置创建SQLite数据库：

```
%LOCALAPPDATA%\MedicalProgress\medical_progress.db
```

完整路径示例：
```
C:\Users\您的用户名\AppData\Local\MedicalProgress\medical_progress.db
```

## 🎯 使用步骤

### 1. 启动应用
双击 `MedicalProgress.App.exe` 启动程序

### 2. 新建患者
- 点击左侧"新建患者"按钮
- 填写患者基本信息：
  - 姓名
  - 性别
  - 年龄
  - 入院日期
  - 主诉（必填）
  - 现病史
  - 查体
  - 辅助检查
  - 治疗方案
- 点击"保存并生成病程"

### 3. 生成病程记录
- 从左侧患者列表选择患者
- 选择模板类型：
  - **首次病程**：入院首日完整病程
  - **日常病程**：后续每日记录
  - **出院小结**：出院时总结
- 点击"生成病程"

### 4. 自动重复检测
- 系统自动检测20字以上重复
- 如有重复，自动修正并提示
- 可手动点击"检测重复"查看详情

### 5. 复制使用
- 点击"复制文本"按钮
- 粘贴到您的病历系统中

### 6. 保存记录
- 点击"保存记录"保存到本地数据库

## ⚙️ 功能说明

### 核心功能
| 功能 | 说明 |
|------|------|
| 患者管理 | 新建、编辑、删除患者信息 |
| 病程生成 | 基于模板自动生成标准病程 |
| 重复检测 | 检测并修正20字以上重复内容 |
| 历史记录 | 保存和查看历史病程记录 |
| 文本导出 | 一键复制到剪贴板 |

### 重复检测算法
- 使用N-gram分词技术
- 检测连续20字以上的重复
- 自动提供修改建议
- 支持手动修正

## 🔧 常见问题

### Q1: 提示缺少DLL文件？
**A**: 请使用"发布.bat"重新编译，确保生成自包含版本

### Q2: 数据库文件在哪里？
**A**: `%LOCALAPPDATA%\MedicalProgress\medical_progress.db`

### Q3: 如何备份数据？
**A**: 复制上述数据库文件即可备份所有患者和病程数据

### Q4: 如何删除所有数据？
**A**: 删除数据库文件即可（需要先关闭应用程序）

### Q5: 生成的病程格式不对？
**A**: 请检查输入的患者信息，确保主诉、病史等内容填写完整
